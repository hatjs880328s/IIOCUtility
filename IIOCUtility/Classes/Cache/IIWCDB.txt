由于Cocoapods编译冲突问题，使用动态库引入WCDB
(1) 获取WCDB源码，并更新sqlcipher的submodule。
git clone https://github.com/Tencent/wcdb.git
cd wcdb
git submodule update --init sqlcipher
(2) 将WCDB.xcodeproj添加到工程中.
(3) 在项目工程配置的General->Enbedded Binaries中添加WCDB.framework。注意这里有两个WCDB.framework，要选择动态库那个。选完后可在Build Phases->Target Dependencies中确认，若为WCDB则正确，而WCDB iOS Static则是静态库使用的。

1.IIDatabase类
z.对MODEL进行 CRUD 等基础操作
a.所有调用database此工具类的文件必然是DAL层
b.model需要按照WCDB定义进行格式化
c.如果需要新的wcdb-api需要自行添加到database类中

2.IIDatabaseManager类
z.只提供关闭数据库的功能
a.所有调用Manager此工具类可能包含在业务代码中-（退出登陆、切换企业）
b.此类只有关闭方法，为了屏蔽cpp引起文件名字修改的问题

3.数据库技术切换
a.目前将所有的调用方法限制在DAL中，以后替换数据库只需要更改 database & databasemanager 类，如果这俩类无法进行技术替换，则需要将dal中对model的CRUD进行修改
b.Model和wcdb耦合较深，需要替换



注意
1.如果需要关闭数据库之后操作数据库文件（如移动等），需要使用异步的关闭方法
